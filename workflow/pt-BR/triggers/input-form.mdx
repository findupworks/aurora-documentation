## Visão Geral

Use um disparador de formulário de entrada quando um fluxo de trabalho deve ser iniciado a partir do editor com campos tipados. O painel de execução mostra exatamente os campos que você define, então o fluxo de trabalho sempre recebe dados limpos.

## O que você define

Adicione campos no construtor de formato de entrada (texto, número, booleano, JSON, etc.). Para cada campo:

- O valor aparece como `<blockName.field>` no fluxo de trabalho.
- A carga completa é refletida em `<blockName.input>` para maior conveniência.

Se você deixar o formulário vazio, o disparador não tem saídas.

Quando você pressiona Executar no editor, o painel mostra o formulário. Os valores enviados alimentam diretamente a saída do disparador, então os blocos posteriores podem referenciá-los sem análise adicional. Os números são convertidos para números, os booleanos são convertidos para verdadeiro/falso, e os campos JSON são analisados antes que o fluxo de trabalho os veja.

## Fluxos de trabalho secundários

Os disparadores de formulário de entrada também alimentam o bloco de fluxo de trabalho. Quando você incorpora um fluxo de trabalho secundário, a etapa de mapeamento mostra os campos do formulário secundário para que você possa conectar variáveis do pai. Tudo o que você mapeia se torna o envio do formulário de entrada do fluxo de trabalho secundário.
